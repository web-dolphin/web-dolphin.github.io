5.2.2 ECMAScript原型与原型链(一)

1. 文档

虽然ES6虽然引入了class关键字，但是，JavaScript关于类的概念任然是基于原型。

JavaScript关于属性和方法的复用和继承，都是通过对象，通过实例对象实现对构造函数的属性和方法的复用。

1. Constructor: 构造函数

  1. 函数是JavaScript的一等公民，构造函数与普通函数并没有什么区别，只有当函数通过new关键字调用时，才称之为构造函数；

  2. 通常将构造函数的首字母大写，以显示与普通函数的区别；

  • new关键字调用便创建了实例对象；

  • Person就是一个构造函数

  • person就是构造函数Person创建的实例对象

2. Constructor.prototype: 构造函数的原型对象

  1. 构造函数的原型对象： Constructor.prototype

  • JavaScript的构造函数实例化一个对象的同时，具有一个属性： prototype, 称之为构造函数的原型对象,

  • Constructor.prototype： 原型对象

  1. Constructor.prototype具有两个属性： constructor和__proto__

  • 其中constructor指向了构造函数本身；

  • __proto__：是构造函数Function.prototype(原型对象)的原型对象，指向创建它的上级构造函数，本例实际就是指向构造函数Function的原型对象；

  • 该原型对象(Person.prototype)也有一个自己的原型对象(__proto__) ，层层向上直到一个对象的原型对象为 null

    ``js
    const array = new Array()
    console.log(array.proto === Array.prototype); // true
    console.log(Array.prototype.proto === Object.prototype); // true
    console.log(Object.prototype.proto); // null

4. 构造函数与实例对象的关系

  • 通过以上的推论，可以看出构造函数在实例化一个对象的时候，同时创建了一个原型对象(prototype);

  • 该原型对象具有两个属性： constructor和__proto__;

  • 其中constructor返回构造函数本身；

  • __proto__指向实例化自身的构造函数的原型对象.

  • Array包装函数

  • 构造函数

5. 原型链

  • JavaScript 对象有一个指向一个原型对象的链,当试图访问一个对象的属性时，它不仅仅在该对象上搜寻，还会搜寻该对象的原型，以及该对象的原型的原型，依次层层向上搜索，直到找到一个名字匹配的属性或到达原型链的末尾;

6. 总结

  1. JavaScript的任何函数都是有Function构造函数实例化而来；

  • 任何函数都是Function的实力对象

  • JavaScript的内置对象，本质上都是构造函数；

  • Math对象是个例外，它是Object的实例对象

  • Math对象的原型链示例

  1. Math只是Object的实力对象，本身不是构造函数，所以没有prototype原型对象；

  2. Object.prototype.__proto__=null,说明Object处于原型链的顶层；

  • Array为例演示原型链的关系

  1. Array原型对象是Object构造函数的实例;

  2. Array本身也是一个构造函数，它实例化了一个array;

7. 扩展

  1. 目的： 通过原型链的关系了解实例对象与原型之间的关系

  • 创建一个Person构造函数

    ``js
    function Person({name,age,gender}) {
    this.name = name,
    this.age = age,
    this.gender = gender
    }

  • 实例化对象

    ``js

    console.log(person);

  • prototype本身也具有一个原型对象__proto__，指向其上一级的构造函数Object的原型对象prototype;

  • prototype创建了一个favorite属性，通过这个原型链关系，可以访问原型链上的所有属性；

  • 检查Object.prototype.__proto__是否存在上一级构造函数的原型对象？ null： 表示没有；

5.2.3 ECMAScript原型与原型链(二)

1. 原型与原型链扩展

1. 构造函数

  • 构造函数本质是函数，只是通过new关键调用时，才构成构造函数；

  • 通常情况下为了区别普通函数，常常将构造函数的首字母大写；

  • 构造函数实例化对象时，同时创建了自己的原型对象： prototype，同时也为其实例对象创建了一个原型对象： __proto__；

  • 实例对象的原型对象(__proto__)的引用地址与创建自己构造函数原型对象(prototype)的引用地址相同, 因此;

  • 上述特性表明：

    1. 在构造函数原型对象上添加任何属性，其实例对象的原型对象(__proto__)同样可以访问和共享，因为他们原本指向共同的内存；

    2. 在其实例对象上添加任何属性，其构造函数原型对象(prototype)同样可以访问和共享，因为他们原本指向共同的内存；

  • 构造函数同样存在一个原型对象(__proto__),其引用地址：创建它的构造函数(Function)的原型对象(prototype)

  • 构造函数Person的原型对象(prototype),存在两个属性：

    1. constructor: 其指向构造函数Person本身；

    2. __proto__: 其指向了创建构造函数Person的原型对象(prototype)的构造函数： Object的原型对象prototype，也就是说： Object是Person的原型对象(prototype)的构造函数；

    3. 这样，Object.prototype 与 Person.prototype.__proto__;Person.prototype 与 person.__proto__之间构成了一个原型链;

  • 请打开浏览器开发者工具： 仔细查看：Person.prototype,person.__proto__

  • person.__proto__与Person.prototype

5.1.1 面向对象的程序设计

1.JS面向对象

  • 面向对象(OOP:Object Oriented)语言的标志: 类的概念，通过类可以创建任意多个具有相同属性和方法的对象；
  • ECMAScript中没有类的概念，因此JS对象也与基于类语言的对象有所不同；

2.ECMAScript关于对象的定义

  • ECMA-262 把对象（object）定义为“属性的无序集合，其属性可以包含基本值、对象或函数”；
  • 严格来说，对象是无特定顺序的值的数组；
  • 对象的属性和方法都映射到一个值，称之为键值对，其中属性和方法称之为键，映射值可以是基本数据，也可以是函数；
  • 每个对象都是基于一个引用类型创建的。这个引用类型可以是原生类型，也可是开发人员定义的类型；

3.ECMAScript 类的概念

  • 每个对象都由类定义；
  • 可以把类看做对象的配方；
  • 类不仅要定义对象的接口，还要定义对象的内部工作(使属性和方法发挥作用的代码);
  • 编译器和解释程序都根据类的说明构建对象;
  • 接口（interface）: 开发者访问的属性和方法;
  • 程序使用类创建对象时，生成的对象叫作类的实例
  • 对类生成的对象的个数的唯一限制来自于运行代码的机器的物理内存；
  • 每个实例的行为相同，但实例处理一组独立的数据；
  • 由类创建对象实例的过程叫做实例化。

4.ECMAScript 对象类型

  • ECMA-262 把本地对象（native object）定义为“独立于宿主环境的 ECMAScript 实现提供的对象”；
  • 简单来说，本地对象就是 ECMA-262 定义的类（原生类型）。它们包括：
    • Object
    • Function
    • Array
    • String
    • Boolean
    • Number
    • Date
    • RegExp
    • Error
    • EvalError
    • RangeError
    • ReferenceError
    • SyntaxError
    • TypeError
    • URIError

5.面向对象语言的要求

  • 面向对象语言具有四种基本能力：

    • 封装 - 把相关的信息（无论数据或方法）存储在对象中的能力；
    • 聚集 - 把一个对象存储在另一个对象内的能力；
    • 继承 - 由另一个类（或多个类）得来类的属性和方法的能力；
    • 多态 - 编写能以多种方法运行的函数或方法的能力；
    • ECMAScript 支持这些要求，因此可被是看做面向对象的。

6.对象的构成

  • 在 ECMAScript 中，对象由特性（attribute）构成，特性可以是原始值，也可以是引用值;
  • 如果特性存放的是函数，它将被看作对象的方法（method）；
  • 否则该特性被看作对象的属性（property）。

6.2 prototype原型方式的缺点

  • 首先，这个构造函数没有参数；

  • 不能通过给构造函数传递参数来初始化属性的值；

  • 现实中对象却很少被多个实例共享属性；

7.混合模式创建对象

  • 构造函数模式+原型模式创建对象；

  • 混合模式共享着对相同方法的引用，又保证了每个实例有自己的私有属性。最大限度的节省了内存;

8. 动态原型模式

  • 有些对象会有方法(动作、行为)，而这些方法都是函数，如果把这些方法和函数都放在构造函数中声明就会导致内存的浪费

9.5 Flex弹性布局

1. Flex

1 . main-axis(主轴)

2. cross-axis(交叉轴)

3. flex-direction:

  • Flexbox Container

    Attribute Description
    row main-axis(主轴)是 row
    column main-axis(主轴)是 column
    row-reverse main-axis(主轴)是 row,逆序排列
    column-reverse main-axis(主轴)是 column,逆序排列

4. justify-content 和 align-items

  • justify-content ： 主轴排列方向

  • align-items：交叉轴排列方向

    Attribute Description
    center
    flex-start
    flex-end

5. flex-wrap

  • 定义是否换行

    Attribute Description
    nowrap 不换行
    wrape 换行

6. flex-flow

  • flex-direction、jflex-wrap 的简写

7. align-content

  • 当 flex-wrape：wrape 时，生效；用于行于行之间的对齐方式

    Attribute Description
    center
    flex-start
    flex-end
    initial
    space-around
    space-evenly
    space-between

2. Flexbox Item

1. order

  • 默认值： 0
  • 数值越大越排在后
  • 可以取负值

2. align-self

  • 可以用于改写 flex container 中，align-items 中的设定值
  • 属性值与 flex container 相同

3. flex-basis: 好像没有效果

  • 是设定 flex-item 的主轴方向上的大小

    Attribute Description
    0 width: 0px;
    100px 原值 + 100px
    auto 原值

4. flex-grow

当主轴方向有剩余空间时，flex-item 沿主轴方向扩展的设定,flex-grow: 0；表示保持原始值，不放大；

5. flex-shrink

当主轴方向空间不够时，flex-item 沿主轴方向缩小的设定,flex-shrink: 0；表示保持原始值；

6. Flex

Flex Item 项目中： flex-grow + flex-shrink + flex-basis 的简写形式

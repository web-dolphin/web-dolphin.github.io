第一章 2 JS数据类型

1. 基本数据类型

1.1 Number

  1. JS和其他程序设计语言(如C和 Java)的不同之处，在于它并不区别整型数值和浮点型数值;

  2. 在 JS`中，所有的数字都是由浮点型表示；

  3. JS所能表示的整数范围： - 2 ^53^ ~ 2 ^53^

  4. 数字直面量(numeric literal)： 一个数字直接出现在 JS程序中，该数字就称为数字直面量；

    1.1.1 整型直面量

    • 二进制数字： 0 1

      例如： 1111 1111 = 1 x 2 ^7^ + 1 x 2 ^6^ + 1 x 2 ^5^ + 1 x 2 ^4^ + 1 x 2 ^3^ + 1 x 2 ^2^ + 1 x 2 ^1^ + 1 x 2 ^0^ = 255 (十进制)

    • 十进制数字： 0 ~ 90

      例如： 192

    • 八进制数字： 0 ~ 7

      例如： 0377 = 3 x 8 ^2^ + 7 x 8 ^1^ + 7 x 8 ^0^ = 255 (十进制)

    • 十六进制数字： 0 ~ 9 a b c d e f

      例如： 0xff = 15 x 16 ^1^ + 15 x 16 ^0^ = 255 (十进制)

    1.1.2 浮点型直面量

    1. 浮点型直面量表示方法：

      • 实数计数法： 有整数，小数点，小数组成的数

        例如： 3.14，.333

      • 指数计数法：

        例如： 6.02e23 = 6.02 x e ^23^ 或者 6.02 x E ^23^

    1.1.3 JS 溢出

    1. 溢出： overflow，当数字运算结果超过了 JS 所能表示的数字上限，称之为溢出，结果以： Infinity 表示；

    2. 下溢： underflow，当数字运算结果超过了 JS 所能表示的负数下限时，称之为下溢，结果以：--Infinity 表示；

    3. JS 在溢出，下溢或被零整除时不会报错

    4. 零除以零是没有意义的，结果为： not-a-number， 用NaN表示

      例如：

    1.1.4 NaN

    1. JS 中的非数字值有些特殊，它和任何数值都不相等，包括它自己；

    2. 设置两个数不相等： x != x;

    3. 判断一个数是 NaN：isNaN()， 当参数是 NaN 或者是非数字值(例如： 字符串或对象)，则返回： true；

    4. 任意数除以 0 === NaN

    1.1.5 负零值

    1. 负零值具有同样特殊性，负零值 = 正零值；

    1.1.6 二进制浮点数和四舍五入错误

    1. JS 采用 IEEE-754 浮点表示法，这是一种二进制表示法，可以精确地表示分数，例如： 1/2，1/98 等；

    3. 数值的范围

    • Number.NEGATIVE_INFINITY: 负无穷
    • Number.POSITIVE_INFINITY: 正无穷
    1.1.7 数值转换
    1. false、null、空字符串、空数组转换后 = 0
    2. true 转换后 = 1
    3. {}、undefined、非数字字符串、NaN、非空数组转换后 = NaN

1.2 String

  • 字符串是一组由 16 位值组成的不可变的有序序列；
  • 每个字符通常来自于 Unicode 字符集；
  • JS 通过字符串类型来表示文本；
  • 字符串的长度(length)是其所含 16 位值的个数
  • JS 字符串的索引从零开始；
  • 空字符串的长度为零

1.2.1 字符串直接量

  • 字符串直接量： 由单引号或者双引号括起来的字符序列；

  • 由双引号界定的字符串，可以包含单引号；

  • 由单引号界定的字符串，可以包含双引号；

  • 字符串直接量必须写在一行；

  • 字符串直接量可以拆分成多行，每行必须以反斜线( \ )结束；

  • 反斜线和行结束符都不算是字符串直接量的内容；

  • 如果希望在字符串直接量中另起一行，可以使用转义字符( \n )

1.2.2 转义字符： escape sequence

  1. 在 JS 字符串中，反斜线后加一个字符，就不再表示其字面含义；

    转义字符串 说明
    \n 换行符
    \ ‘ 单引号
    \ “ 双引号
    \ b 退格
    \ xA9 版权符
    \ t 水平制表符
    \ v 垂直制表符
    \ f 换页符
    \ r 回车符
    \ \ 反斜线

1.2.4 模式匹配

  1. JS 定义了 RegExp()构造函数，用来创建表示文本匹配模式的对象。这些模式称为“正则表达式”(regular expression),JS 采用了 Perl 中的正则表达式语法，String 和 RegExp 对象均定义了利用正则表达式进行匹配和查找与替换的函数；

  2. RegExp并不是JS的基本类型，但是，它们依然具有直面量写法，可以直接在 JS` 中使用。在两行斜线之间的文本构成了正则表达式直接量，第二个斜线之后也可以跟随一个或多个字母，用以修饰匹配模式的含义；

  3. RegExp 定义了很多有用的对象，字符串同样具有可以接受 RegExp 参数的方法

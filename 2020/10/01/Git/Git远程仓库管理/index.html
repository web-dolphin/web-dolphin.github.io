Git远程仓库管理

什么是 Git

  • Git： 是一个开源的分布式版本控制系统，本质上是一个命令行版本管理软件；

本地仓库

1 本地项目组成

  • 包括： Workspace(工作区)、Staging Area(暂存区)、Local Repository(本地仓库)

2 工作原理

  • Git 将 Workspace 的文件通过添加(add)操作到 Staging Area，
  • 如果确认不需要修改时就将项目提交(commit)到 Local Repository
  • 如果确认不需要修改时就将 Local Repository 推送(push)到远程仓库
  • 本地操作包括：初始化(init)仓库、添加(add)、提交(commit)、状态(state)查询、创建分支(branch)、分支合并(merge)、分支切换(checkout)、日志(log)查询等；

3 常用命令

  • 添加(add)

  • 提交(commit)

  • 状态(state)查询

  • 创建分支(branch)

  • 分支合并(merge)

  • 分支切换(checkout)

  • 日志(log)查询等

  • 初始化项目

Github 远程仓库

  • Github: 本质上是远程服务器仓库(Remote Repository)；

1 Github 公/密钥

  • git 与 Gthub 账号关联式以后，两者之间采用了 https 通信协议，在拉取(pull)项目到本地，或者推送(push)项目到 Github 远程仓库时，git 必须每次输入密钥，操作比较繁琐；

  • 为了减少操作不便，可以在 Github 仓库，创建本地密钥对，并在远程仓库设置公钥，git 克隆文件时，使用 SSH 通信协议
